## Detailed Changes

- **getMatchesForRfq()**: Was returning `{extractedItemId, description}[]` — MatchReviewPanel was crashing at runtime accessing `g.extractedItem.id` (undefined). Now returns `RfqMatchGroupView[]` with full embedded `RfqExtractedItemView`.
- **grandTotal computation**: Backend now ignores client-provided grandTotal. `computeGrandTotal()` calculates: sum subtotals → apply order discount → apply tax → round to 2dp. Test assertion updated from 1200 (client-provided) to 1600 (server-computed from 20×40 + 50×16).
- **email template**: Removed `/ 100` from all price display lines. Prices are now stored as dollars, not cents.
- **attachment parsing**: `aggregateAttachmentContent()` now returns `bool` (true if all attachments settled). After `attachment_parse` job, if `aggregate()` returns true, handler enqueues `rfq_extract` job. Fixes race condition where extraction fired before attachments parsed.
- **intake classification**: Gmail and Slack connectors previously called `createRfqFromIntake()` directly, bypassing LLM classification. Now both call `classifyAndIntake()` which runs classify first.
