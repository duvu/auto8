## Detailed Changes

### WhatsApp Connector
- `WhatsappController`: `GET /webhooks/whatsapp` challenge, `POST /webhooks/whatsapp` process
- `WhatsappConnectorService`: HMAC via `X-Hub-Signature-256`, `processWebhook()`, `downloadMedia()` (16MB limit), `testConnector()` via Graph API
- Dedup via `slackMessageId` field storing WhatsApp message ID

### Telegram Connector
- `TelegramController`: `POST /webhooks/telegram/:secret`
- `TelegramConnectorService`: secret-in-URL auth, `processUpdate()`, `downloadDocument()` (20MB limit), `testConnector()` via getMe
- Dedup key = `tg_{connectorId}_{message_id}`

### Zalo OA Connector
- `ZaloController`: `GET /webhooks/zalo/:connectorId` challenge, `POST /webhooks/zalo/:connectorId` process
- `ZaloConnectorService`: HMAC-SHA256 via `mac` field, verifyToken challenge, attachment download (20MB), `testConnector()` via getoa
- Dedup key = `zalo_{connectorId}_{msg_id}`

### Email Reply Threading
- `GmailService.syncMessages()`: thread detection тАФ checks for existing intake with same `gmailThreadId`, sets `isReply=true`, `replyToRfqId`
- `RfqIntakeService.createRfqFromIntake()`: persists `isReply`, `replyToRfqId`; skips `rfq_extract` job when `isReply === true`
- `listRfqs()`: `includeReplies = false` default; `getReplies(rfqId)` added
- New endpoint: `GET /rfqs/:rfqId/replies`

### Outbound Webhooks
- `WebhookEndpointService`: CRUD, encrypt/decrypt secret, `findEnabledForEvent()`, test with HMAC ping
- `WebhookDeliveryService`: `OnModuleInit` registers `webhook_deliver` handler; 3-attempt retry (30s/5m backoff)
- `WebhookEmitterService`: `emit(event, payload)` тЖТ queries enabled endpoints тЖТ enqueues per endpoint
- Events: `rfq.created`, `quote.approved`, `quote.sent`
- `WebhookEndpointController`: `POST/GET/PATCH/DELETE /webhooks/endpoints` + test endpoint
